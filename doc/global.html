require(namespace) → {Module#exports}

Implementation require for XScript. See README.
Parameters:
Name Type Description
namespace string Пространство имен модуля или абсолютный путь к JS-файлу. В последнем случае require воспринимает это как обращение к компоненту модуля (см. диаграмму алгоритма работы в исходниках).
Properties:
Name Type Argument Default Description
path array Массив путей с дирректориям, в которых будут искаться файлы модулей. Каждый путь представляет собой уровень переопределения, выстраивающий соответствую цепочку наследования модулей из одного пространства имен.
extension string <optional>
 'js' Расширение файлов модулей. См Module#load.
packageName string <optional>
 'package.json' Имя JSON-файла, содержащий информацию о модуле. См Module#load.
prototype object Общий прототип объектов API модулей (Module#exports).

Example
// Загрузка модуля из разных уровней переопределения
require.path = ['/path/to/first/dir/', 'path/to/second/dir'];
require('names.space');
// В этом случае объект API модуля, загруженного из файла "path/to/second/dir/names.space.js" будет наследовать от API модуля, загруженного из "/path/to/first/dir/names.space.js".

// Доступ к модулям из пространства имен
require('some.name.space');
$XM.some.name.space; // объект Module#exports.
